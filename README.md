# Data Processing and Analysis Project\n\nThis project provides a robust and automated solution for processing tabular data, specifically from Excel files, and generating structured JSON output. It leverages Python with Pandas for data manipulation and GitHub Actions for continuous integration and deployment, ensuring reliable and up-to-date insights.\n\n## Project Overview\n\nThe core of this project revolves around:\n1.\t**Data Ingestion**: Starting with an Excel spreadsheet (`data.xlsx`).\n2.\t**Data Transformation**: Converting the Excel data into a more manageable CSV format (`data.csv`).\n3.\t**Data Analysis**: Executing a Python script (`execute.py`) to perform specific data processing and aggregation tasks.\n4.\t**Result Generation**: Outputting the processed data into a JSON file (`result.json`).\n5.\t**Automation**: Automating the entire pipeline using GitHub Actions, including code quality checks (Ruff) and publishing results via GitHub Pages.\n\n## Project Structure\n\n```\n.\n├── .github/\n│\t└── workflows/\n│\t\t└── ci.yml\t\t\t# GitHub Actions workflow for CI/CD\n├── data.xlsx\t\t\t# Original Excel data file\n├── execute.py\t\t\t# Python script for data processing\n├── index.html\t\t\t# Responsive front-end for project overview (if deployed)\n├── LICENSE\t\t\t# Project license (MIT)\n├── README.md\t\t\t# This README file\n├── data.csv\t\t\t# (Generated by CI) CSV version of data.xlsx\n└── result.json\t\t\t# (Generated by CI) Output of execute.py\n```\n\n## Setup and Local Execution\n\n### Prerequisites\n\n*\tPython 3.11+\n*\tPandas 2.3+\n*\tRuff (for linting)\n\nYou can install the required Python packages using pip:\n\n```bash\npip install pandas ruff openpyxl\n```\n`openpyxl` is needed for pandas to read `.xlsx` files.\n\n### Running the Script\n\n1.\t**Prepare Data**: Ensure `data.xlsx` is in the project root.\n2.\t**Convert to CSV** (Optional, but done in CI):\n\t```python\n\timport pandas as pd\n\tdf = pd.read_excel('data.xlsx')\n\tdf.to_csv('data.csv', index=False)\n\t```\n\t*Note: The `execute.py` script itself might handle reading `.xlsx` or `.csv`. For consistency with the CI which converts to `.csv` first, we assume `execute.py` might read `data.csv`.*\n\n3.\t**Execute Analysis**:\n\t```bash\n\tpython execute.py > result.json\n\t```\n\tThis command will run the data processing script and save its standard output to `result.json`.\n\n### Code Quality\n\nWe use [Ruff](https://beta.ruff.rs/docs/) for code linting and formatting. To check your code locally:\n\n```bash\nruff check .\n```\nTo fix auto-fixable issues:\n\n```bash\nruff check . --fix\n```\n\n## Continuous Integration and Deployment (CI/CD)\n\nThe project utilizes GitHub Actions for an automated workflow defined in `.github/workflows/ci.yml`.\n\n### Workflow Steps:\n\n1.\t**Checkout Code**: Fetches the repository content.\n2.\t**Setup Python**: Configures the Python environment (3.11).\n3.\t**Install Dependencies**: Installs `pandas`, `ruff`, and `openpyxl`.\n4.\t**Run Ruff Linter**: Checks the Python code for style violations and potential errors. Results are displayed in the CI logs.\n5.\t**Convert data.xlsx to data.csv**: Transforms the input Excel file into a CSV format, making it ready for processing.\n6.\t**Execute Analysis**: Runs `execute.py` and redirects its output to `result.json`.\n7.\t**Upload Artifact**: Stores `result.json` as a build artifact.\n8.\t**Deploy to GitHub Pages**: Publishes the generated `result.json` to GitHub Pages, making it publicly accessible. The `index.html` file is also deployed as a static site.\n\n### Accessing Results\n\nAfter a successful CI run on the `main` branch, the `result.json` file will be available via GitHub Pages at a URL similar to `https://<YOUR_GITHUB_USERNAME>.github.io/<YOUR_REPO_NAME>/result.json`.\nThe `index.html` dashboard will be available at `https://<YOUR_GITHUB_USERNAME>.github.io/<YOUR_REPO_NAME>/`.\n\n## License\n\nThis project is licensed under the MIT License.
